# Authorship Proof

This page documents the methodology for proving sole authorship of the Cemini
Financial Suite codebase for purposes of intellectual property transactions.

---

## Why Authorship Matters

When transferring self-created intellectual property, the author may be eligible
for **long-term capital gains treatment** under **IRC Section 1235**, provided
the property was created through the author's "personal efforts" rather than
purchased or created by employees.

To support this treatment, Cemini includes cryptographically verifiable authorship
proof derived from the git repository's commit history.

---

## Methodology

The `scripts/authorship_proof.py` script:

1. Reads the complete git commit log via `git log --format="%H|%an|%ae|%ai|%s" --all --no-merges`
2. Separates human commits from bot commits (filters `github-actions[bot]` and similar)
3. Computes total commits, date range, and author breakdown
4. Generates a weekly activity histogram
5. Writes a formal authorship statement referencing Section 1235

---

## How to Verify

### Generate the authorship proof

```bash
cd /opt/cemini
python3 scripts/authorship_proof.py
# Outputs: vdr/06_authorship_proof.md, vdr/07_git_stats.json
```

### Independently verify via git

```bash
# All non-bot authors
git log --format="%an" --all --no-merges | \
  grep -v "github-actions\[bot\]" | \
  sort -u

# Commit count by author
git shortlog -sne --all --no-merges | \
  grep -v "github-actions"

# Date range
git log --format="%ai" --all --no-merges | \
  grep -v "github-actions" | \
  sort | \
  head -1 && \
  git log --format="%ai" --all --no-merges | tail -1
```

### Verify via GitHub

The complete commit history is publicly accessible at:
`https://github.com/cemini23/Cemini-Financial-Suite/commits/main`

GitHub displays the verified author identity for each commit, including
cryptographic GPG signatures if configured.
