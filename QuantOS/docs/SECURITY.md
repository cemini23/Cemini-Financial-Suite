# Security Policy

## Supported Versions

Currently, only the latest version of QuantOS™ is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 2.8.x   | :white_check_mark: |
| < 2.8.0 | :x:                |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security-related bugs.**

If you discover a vulnerability that could expose API keys, drain funds, or compromise a User's machine, please report it privately to the author:

- **Contact:** [261469054+cemini23@Users.noreply.github.com](mailto:261469054+cemini23@Users.noreply.github.com)
- **Encryption:** If possible, please use a secure messaging method or clear descriptions to avoid public exposure.

### Our Commitment
We will acknowledge your report within 48 hours and provide a timeline for a patch. We appreciate the community's help in keeping QuantOS™ safe for retail traders.

## Safe Harbor
We will not pursue legal action against researchers who discover and report vulnerabilities responsibly, provided they do not exploit the bug for personal gain or cause harm to others.
